# 라오트러스트(LaoTrust) 기획 요약 및 화면 정의서 정리

> 아래 내용은 **LT-05 기획 최종 요약**과 **LT-04 화면 정의서 v1.0** ODT 문서에서 추출·정리한 것입니다.

---

## 1. [LT-05] 기획 최종 요약 및 막내 인계 지침

**수신:** 개발 요원 막내 (Claude/Cursor)  
**발신:** 전략 사업본부장 장진혁 (수사 총괄 지휘)  
**주제:** 1단계 기획 완료에 따른 개발 가이드라인 및 철학 하달

### Ⅰ. 프로젝트 핵심 정체성 (Core Identity)

| 항목 | 내용 |
|------|------|
| **프로젝트 명** | LaoTrust (라오트러스트) |
| **브랜드 가치** | "신뢰를 기술로 증명하고, 라오스를 연결한다." |
| **비즈니스 모델** | **70:30 하이브리드 전략 (Hybrid Strategy)** |

- **70% (숨고 컨셉):** 고숙련 전문가 견적 매칭  
  - 에어컨 수리, 전기, 가사 도우미 등  
- **30% (알바몬 컨셉):** 위치 기반 단기 구인구직  
  - 식당 서버, 단순 노무 등  

### Ⅱ. 기술 및 운영 표준 (Standard Specs)

| 항목 | 내용 |
|------|------|
| **디자인 규격** | 인디고 블루(신뢰) & 화이트(청결). **AST-01(시연 웹)** UI를 표준 가이드라인으로 사용 |
| **개발 환경** | **Flutter** 기반 크로스 플랫폼 개발 |
| **언어 대응(i18n)** | 모든 텍스트는 외부 JSON 파일로 관리, **라오어/영어/한국어** 즉시 전환 구조 |

### Ⅲ. 막내 요원을 위한 특별 지시 (Instruction for AI)

1. 막내는 라오트러스트 전담 개발 요원으로 본부장 지시를 수행한다.
2. 모든 코딩 시 **'7:3 전략'**과 **'신뢰 인증 시스템'**을 최우선 가치로 둔다.
3. 디자인을 임의로 변경하지 말고, 컨펌된 **인디고 블루** 톤의 전문성을 유지한다.
4. **LT-04(화면 정의서)**를 정밀 분석하여 코딩 설계를 시작한다.

### [참고 용어]

- **숨고 스타일:** 사용자가 요청서를 쓰면 전문가가 견적을 보내는 '역경매/매칭' 방식
- **알바몬 스타일:** 고용주가 공고를 올리면 사용자가 지도를 보고 지원하는 '공고/나열' 방식
- **MVP (Minimum Viable Product):** 최소 기능 제품

## 2. [LT-04] 화면 정의서 v1.0

**문서 번호:** LT-04  
**작성일:** 2026-02-20  
**작성자:** 수사본부 에이스 첫째 (Gemini)  
**기준 자산:** AST-01 (인터랙티브 시연 웹)

---

### 1. 홈 화면: 메인 대시보드 (Home Dashboard)

사용자가 앱을 실행했을 때 마주하는 '중앙 관제 센터'입니다.

| 항목 (Component) | 상세 정의 (Description) | 동작 및 이벤트 (Events) |
|------------------|--------------------------|--------------------------|
| **상단 헤더(Header)** | 인디고 블루(#3F51B5) 배경, LaoTrust 로고 | 클릭 시 홈 화면 최상단으로 이동(새로고침) |
| **알림 아이콘(Bell)** | 우측 상단 종 모양 아이콘 (신규 알림 시 빨간 점 표시) | 클릭 시 신규 매칭 및 메시지 알림 센터로 이동 |
| **통합 검색바(Search)** | '전문가 또는 일자리를 검색하세요' 문구 포함 | 키워드 입력 후 확인 시 통합 검색 결과 페이지 노출 |
| **전문 서비스(70%)** | 에어컨, 가사, 전기, 배관 아이콘 (그리드 형태) | **[중요]** 아이콘 클릭 시 해당 카테고리 **[요청서 작성]** 이동 |
| **급구 알바(30%)** | 가로형 카드 리스트. 제목, 급여, 위치 노출 | 카드 클릭 시 해당 **[알바 상세 내용]** 팝업 노출 |
| **탭 바(Bottom Nav)** | 홈, Jobs(구인), 채팅, 내 정보 (4단 고정 메뉴) | 각 탭 클릭 시 해당 섹션으로 즉시 화면 전환 |

---

### 2. 요청서 화면: 전문가 매칭 플로우 (Request Flow)

숨고(Soomgo) 스타일의 '사전 문진'을 통해 신뢰를 확보하는 단계입니다.

| 항목 (Component) | 상세 정의 (Description) | 동작 및 이벤트 (Events) |
|------------------|--------------------------|--------------------------|
| **진행 표시줄(Bar)** | 상단에 현재 질문 진행률 시각화 (예: Step 1/3) | 질문 응답 후 '다음' 버튼 클릭 시 게이지 상승 |
| **객관식 문항(Choice)** | '바람 안 시원함', '소음' 등 상황별 문항 | 중복 선택 가능. 선택 시 테두리 파란색 강조 |
| **이미지 첨부(Photo)** | 현장 사진 업로드 버튼 (카메라 아이콘) | 클릭 시 스마트폰 갤러리 또는 카메라와 연동 |
| **하단 버튼(Action)** | '다음 단계로' (검정색 대형 버튼) | 클릭 시 다음 질문지로 슬라이딩 전환 |
| **이전 버튼(Back)** | 좌측 상단 뒤로가기 화살표 아이콘 | 클릭 시 이전 질문 또는 홈 화면으로 복구 |

---

### 3. 알바 리스트 화면: 위치 기반 구인 (Job Board)

알바몬(Albamon) 스타일의 '기동성' 중심 인력 매칭 구역입니다.

| 항목 (Component) | 상세 정의 (Description) | 동작 및 이벤트 (Events) |
|------------------|--------------------------|--------------------------|
| **지도 전환(Map)** | 우측 상단 오렌지색 플로팅 버튼 (지도 모양) | 클릭 시 리스트 목록이 **구글 지도** 화면으로 전환 |
| **채용 카드(Card)** | 업체명, 위치, 시간, 급여 정보 집약 표시 | 카드 클릭 시 **상세 내용** 및 지도상 위치 표시 |
| **지원하기(Apply)** | 각 카드 우측 하단 파란색 버튼 | 클릭 시 구인자에게 내 프로필 전송 및 채팅 시작 |
| **필터(Filter)** | 거리순, 높은 급여순, 단기/장기 등 정렬 선택 | 필터 적용 시 실시간으로 리스트 목록 업데이트 |

---

> 본 화면 정의서는 AST-01(시연물)을 기준으로 작성되었으며, 실제 개발 단계에서 막내(Claude)가 세부 스펙을 추가할 예정임.
