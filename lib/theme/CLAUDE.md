# 🏛️ LaoTrust Project Constitution (v2.0)

## 🏗️ Core Identity & Design Vision
- [cite_start]**Identity**: Trust-first lifestyle platform in Laos (Expert services 70 : Quick jobs 30) [cite: 112, 188, 229]
- [cite_start]**Benchmark**: Clean white mood + high-clarity quick job UX [cite: 211, 212]
- **Design principles**
  - [cite_start]**Base**: Use clean **white (#FFFFFF)** backgrounds for openness [cite: 226, 269]
  - [cite_start]**Point**: Use **indigo blue (#3F51B5)** only for trust signals (badges, primary CTAs) [cite: 220, 226]
  - [cite_start]**Scalability**: Prefer atomic, composable UI components for future verticals [cite: 219, 238]
- [cite_start]**Localization**: All user-visible text must be i18n’d (`ko`, `en`, `lo`) and account for Lao text expansion [cite: 189, 258]

## 🛠️ Engineering Non-negotiables (The Iron Rules)
1. [cite_start]**Monorepo & shared models**: Keep Flutter(App) and Firebase(Server) models consistent [cite: 14, 184]
2. [cite_start]**No “vibe coding”**: Design first, then implement [cite: 9, 81, 196]
3. [cite_start]**Token efficiency**: Prefer direct, targeted tooling and searches [cite: 76, 186]
4. [cite_start]**Language policy**: Code in English; keep comments concise and consistent [cite: 202, 259]

## 🚦 Controls (Hooks & Process)
- [cite_start]**PreToolUse**: Get approval before risky or out-of-scope actions [cite: 57, 193]
- [cite_start]**PostToolUse**: After edits, self-check theme colors and i18n key coverage [cite: 60, 194]
- [cite_start]**Rule capture**: Record newly discovered rules for future sessions [cite: 37, 191]

## 🧠 Thinking Hierarchy
- **Normal**: Small UI changes and typo fixes
- **Think Hard**: Feature implementation and schema design
- [cite_start]**Ultra-think**: Complex logic (payments, matching algorithms) when explicitly requested [cite: 88, 161, 197]
